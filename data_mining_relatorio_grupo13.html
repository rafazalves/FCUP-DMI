Data Preparation

Aps analisar os ficheiros fornecidos foi verificado que:

O nico ficheiro que continha missing values era o trans_dev.csv nas colunas operation, k_symbol, bank e account. Para resolver esse problema, e uma vez que a nica coluna que possa ser til para verificar se um loan pode ser fornecido ou no a account, foram colocados os missing values a 0 porque se existem missing values significa que s existe uma pessoa na conta dessa transao por isso a conta do companheiro sendo inexistente fica com o valor 0.

trans_dev$account <- ifelse(is.na(trans_dev$account), 0, trans_dev$account)

As colunas operation, k_symbol, bank da tabela trans_dev foram eliminadas uma vez que no contribuem para verificar se um loan pode ser fornecido ou no.

trans_dev <- trans_dev[, !(names(trans_dev) %in% c("operation", "k_symbol", "bank"))]

Na coluna birth_number da tabela client o valor referente ao ms em que a pessoa nasceu, dependendo do sexo do cliente, encontra-se normal (cliente do sexo masculino) ou ms+50 (cliente do sexo feminino). Para resolver este problema, e visto que o sexo do cliente no tem relevncia para o que se pretende aferir neste projeto, verificamos quais os valores referentes ao ms que eram superiores a 12 e, nesses casos, subtramos 50 ao seu valor para obter o ms correto.

# Converter para character para se conseguir manipular
client <- mutate(client, birth_number = as.character(birth_number)) 
# Funo para transformar birth_number
transform_birth_number <- function(number) {
  if (nchar(number) == 6) {
    # Extrair os dois dgitos do meio (referentes ao ms)
    bb <- as.integer(substr(number, 3, 4))
    
    if (bb > 12) {
      number <- paste(substr(number, 1, 2), sprintf("%02d", bb - 50), substr(number, 5, 6), sep = "")
    }
  }
  return(number)
}
client <- mutate(client, birth_number = sapply(client$birth_number, transform_birth_number))
# Converter 'birth_number' novamente para integer
client$birth_number <- as.integer(client$birth_number)

Na tabela district foi verificado que as colunas no..of.commited.crimes..95 e unemploymant.rate..95 tinha o seu tipo definido como char e na coluna unemploymant.rate..95 existia um valor desconhecido (?). Para resolver estes problemas foram efetuadas as seguintes operaes: Para a coluna no..of.commited.crimes..95 o seu tipo foi passado para int;

district$no..of.commited.crimes..95 <- as.integer(gsub("[^0-9]", "", district$no..of.commited.crimes..95))

Para a coluna unemploymant.rate..95 o seu tipo foi passado para num e o valor desconhecido foi definido com a mdia de unemploymant.rate..95 de todos os distritos.

# Transformar valores para o tipo numeric
district$unemploymant.rate..95 <- as.numeric(as.character(district$unemploymant.rate..95))
# Calcular a mdia de todos os distritos e substituir o valor desconhecido

district$unemploymant.rate..95[is.na(district$unemploymant.rate..95)] <- mean_value

Na tabela loan_final esto presentes os seguintes dados:

  • loan_id, id de todos os loans feitos
  • client_id, id de todos os clientes que pediram um loan
  • client_age, idade do cliente quando pediu o loan
  • amount, quanto foi o loan pedido
  • duration, quanto tempo, em meses, o cliente tem para pagar o loan de volta
  • payments, quanto que o cliente tem de pagar em cada ms
  • average.salary, salrio mdio do distrito onde o cliente vive
  • unemploymant.rate, taxa de desemprego do distrito onde o cliente vive
  • balance, quanto o cliente tinha na sua conta quando pediu o loan
  • status, varivel que indica se o loan foi bem sucedido ou no. Esta varivel a Target Variable que se deseja prever com base em outras colunas do conjunto de dados.

Aps definir o data set final foram considerados diferentes modelos de classificao:

O primeiro modelo a ser considerado foi o Random Forest Model, tendo sido feito da seguinte forma:

set.seed(123)
train_indices <- sample(1:nrow(loan_final), 0.7 * nrow(loan_final))
train_data <- loan_final[train_indices, ]
test_data <- loan_final[-train_indices, ]

rf_model <- randomForest(status ~ client_age + amount + duration + payments + average.salary + unemploymant.rate + balance, data = train_data)

predictions <- predict(rf_model, test_data)

Foi feita uma diviso dos dados num conjunto de treino (70%) e num conjunto de teste (30%), e foi aplicado um modelo Random Forest aos dados de treino que previu os dados de teste. Por fim, para se avaliar o modelo foi mostrada a sua confusion matrix e foi calculada a preciso e a AUC do modelo.
